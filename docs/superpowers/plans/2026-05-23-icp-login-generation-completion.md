# Magick Box ICP Login Generation Completion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the isolated ICP-hosted Magick Box prototype support a real browser path where a user signs in, submits a prompt, runs an AI worker, stores the generated result in ICP media storage, and sees the completed job.

**Architecture:** Keep certified assets, account state, credits, payment intents, job records, media manifests, and generated bytes on ICP canisters. Keep model inference outside ICP behind an explicit local/remote worker service, then write the output back to ICP through the authenticated browser actor.

**Tech Stack:** React/Vite/TypeScript, Motoko persistent actors, icp-cli local gateway, Internet Identity/local browser identity, ICRC payment intent proof, Playwright, Vitest, Node local worker service, Ollama/OpenAI-compatible/MagickAI adapters.

---

## File Structure

- Modify `src/icp/magickboxClient.ts`: add media canister actor creation and keep II URLs compatible with the local gateway.
- Create `src/icp/browserWorker.ts`: browser-safe worker execution client and result validation.
- Modify `src/icp/MagickBoxIcpContext.tsx`: preserve the active identity, execute the selected worker, upload output to `magickbox_media`, attach the manifest to `magickbox_core`, and refresh completed jobs.
- Create `scripts/local-worker-service.mjs`: CORS-enabled local worker HTTP service using the existing worker adapters.
- Create `scripts/smoke-local-icp-ui-generation.mjs`: browser smoke that logs in with local browser identity, generates content, stores it on ICP media, and screenshots the completed UI.
- Modify `package.json`: add local worker and UI generation smoke commands.
- Add/update Vitest tests around browser worker contracts, media actor availability, and no-production-storage guarantees.
- Modify `docs/progress/magickbox-prototype-progress.md`: checkpoint every major result.
- Update handoff/eval docs after verification.
